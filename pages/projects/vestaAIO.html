VestaAIO Sneaker Bot
VIEW CODE VIEW PROJECT
Sneaker reselling is a side-hustle that involves purchasing very limited items then selling them on second hand markets for huge markups. These limited items can go out of stock in as little as 30 seconds, this is due to the existance of sneaker bots. Sneaker bots use a variety of methods to automate the checkout proccess as fast as possible to secure the most profit for their users.

VestaAIO was my attempt at creating a sneaker bot, and at our peak we had 120 users. Developed with a peer of mine, we created a large system of software comprised of the following components:

  • CLI Application
  • Authentication System/Server-Side Services
  • Anti-Bot Solutions/Exploits

The CLI application was an application that used HTTP requests to simulate the important steps of the checkout process. It was multithreaded allowing thousands of "tasks" to attempt checking out at the same time. Additionally, users could save payment information and HTTP proxies locally. A GUI was designed and coded using ReactJS, but was never fully integrated with the backend

The authentication system consisted of two parts, the License API and a Discord bot to make interacting with it simple. Each user was given a software license, they would then use the Discord bot to bind their license to their Discord accounts. After the license was binded, the users would gain access to the rest of the Discord server, they could then use this key to authenticate their current IP address to use the CLI Application.

The final and most complicated aspect of this project was the anti-bot solutions and exploits. The anti bot solutions are covered in the Akamai and Recaptcha pages. As for exploits, during the lifespan of this project, we found a handful of zero day vulnerabilities that allowed us to make our bot more efficent. These exploits were found through many methods including, request fuzzing, URL path transversal, code injection, XXS, and DNS interactions.
